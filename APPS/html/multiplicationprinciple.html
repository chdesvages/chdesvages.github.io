Section 15.1 The Multiplication Principle

Suppose I toss a coin twice: the total number outcomes is 4, since there are 2 outcomes for what the first coin can be (heads or tails), and then 2 outcomes for what the second coin can be. Hence the overall outcomes are (T,T), (T,H), (H,T), and (H,H), of which there are 4.

We can generalise this to events that involve more than just two possible outcomes: if I roll a 6-sided die and then an 8-sided die, the total number of outcomes is \(6\cdot 8=48\text{.}\) I can figure this out because, for each of the outcomes for the first die (either \(1, 2, \dots , \mbox{ or } 6\)), there are 8 possible outcomes for the second die, and so the total of all possible outcomes is \(48\text{.}\) We can generalise this idea using the multiplication principle.

Theorem 15.1.1. Multiplication Principle.

Let \(P\) be a process consisting of \(n\) stages such that at each stage \(i\text{,}\) there are \(a_{i}\) choices we can make, and no two distinct choices can result in the same outcome. Then after \(n\) stages, the total number of outcomes is \(a_{1}\cdot a_{2}\cdots a_{n}\text{.}\)

Proof.

We prove this by induction on \(n\text{.}\) For \(n\in \mathbb{N}\text{,}\) let \(P(n)\) be the claim that, if \(P\) is a process consisting of \(n\) stages such that at each stage \(i\text{,}\) there are \(a_{i}\) choices we can make, then there are \(a_{1}\cdots a_{n}\) outcomes.

  • \textbf{Base case:}.

    If \(n=1\text{,}\) then \(P\) is a process of one stage where there are just \(a_{1}\) choices, and so there are only \(a_{1}\) many outcomes. This proves the base case.

  • \textbf{Inductive step:}.

    Suppose \(P(n)\) is true for some \(n\in \mathbb{N}\text{.}\) Suppose now \(P\) is a process with \(n+1\) stages and the \(i\)th stage has \(a_{i}\) choices. Note that if we stop the process before the last stage, we will have performed a process with \(n\) stages and for each \(i=1, \dots ,n\text{,}\) there are \(a_{i}\) choices. Then the induction hypothesis implies that there are \(a_{1}\cdots a_{n}\) outcomes. However, for each outcome of the first \(n\) stages, there are now \(a_{n+1}\) choices we can make. So the total number of outcomes after the \(n+1\) stage is the total number of outcomes after \(n\) steps times the total number of outcomes that can happen from each of those outcomes, that is \((a_{1}\cdots a_{n})\cdot a_{n+1}\text{.}\) This proves \(P(n+1)\) and finishes the proof.

This seems quite abstract, but let's look at a simple example.

Proposition 15.1.2.

Let \(S\) have \(n\) elements. Then the number of subsets of \(S\) is \(2^{n}\text{.}\)

Proof.

Let \(S\) be a set of \(n\) elements and list the elements as \(S=\{s_{1}, \dots ,s_{n}\}\text{.}\)

Let us define a process for picking a subset \(A\) of \(S\text{:}\) at each stage \(i=1,2,\dots ,n\text{,}\) we decide whether to add \(s_{i}\) to our set or exclude it. At the end of the process, we will have picked a subset of \(A\text{,}\) and every subset of \(A\) is the outcome of such a process. By the Multiplication Principle, since there are \(2\) choices we can make at each stage, there are \(2\cdot 2\cdots 2=2^{n}\) many outcomes for what \(A\) we could have chosen. Thus, there are \(2^{n}\) sets.

If \(S\) is a finite set consisting of exactly \(n\) distinct members, we call \(n\) the cardinality or size of \(S\) and we denote this by \(|S| = n\) (or in some texts by \(\#(S) =n\)).

Definition 15.1.3.

Let \(S = \{a_1, \dots , a_n\}\) be a set of \(n\) distinct objects. An ordering (or arrangement) of \(S\) is a sequence \((a_{1}, \dots ,a_{n})\) in which each element of \(S\) appears exactly once.

For example \((1,3,2)\) and \((3,2,1)\) are orderings of \(\{1,2,3\}\text{.}\) We can use the multiplication principle to count the number of orderings of \(S\text{:}\)

Example 15.1.4.

How many ways are to order the set \(\{1,2,3\}\text{?}\)

What we do is develop a process for constructing all of the orderings. Suppose we would like to choose an ordering \((a_{1},a_{2},a_{3})\) where the \(a_{i}\) are distinct integers in \(\{1, 2, 3\}\text{.}\) We divide the process into 3 stages, where in the \(i\)th stage we pick the number \(a_{i}\) for our ordering.

  • Stage 1:.

    There are \(3\) choices for \(a_{1}\text{:}\) either 1, 2, or 3.

  • Stage 2:.

    Having chosen \(a_{1}\text{,}\) there are now only 2 numbers remaining that we could pick to be \(a_{2}\text{.}\) So there are 2 choices for \(a_2\text{.}\)

  • Stage 3:.

    Having chosen \(a_{1}\) and \(a_{2}\) there is now only one choice for \(a_{3}\text{.}\)

Thus, by the multiplication principle, there are \(3\cdot 2\cdot 1 = 6\) outcomes, that is, \(6\) orderings.

In fact, more generally, we have the following:

Theorem 15.1.5. Ordering Theorem.

Given a set \(S\) of \(n\) distinct objects, there are \(n!\) ways of ordering them.

This is a special case of the following proposition, (part (b) with \(k=m\)).

Proposition 15.1.6.

Let \(S\) be a set with \(n\) elements and let \(0\leq k\leq n\text{.}\)

  1. The number of ways of picking an ordered selection of \(k\) elements from \(S\) (i.e. an ordered list of elements \((a_{1}, \dots ,a_{k})\text{,}\) possibly with repetitions, with \(a_i\in S\)) is \(n^{k}\text{.}\)

  2. The number of ways of picking an ordered selection of \(k\) distinct elements from \(S\) is \(n(n-1)\cdots (n-k+1) = \frac{n!}{(n-k)!}\text{.}\)

Proof.

  1. We count these possibilities using a \(k\)-step process: there are \(n\) options for \(a_1\text{,}\) \(n\) options for \(a_2\text{,}\) and so on. The Multiplication Principle tells us that there are \(n\cdot n\cdots n = n^{k}\) outcomes.

  2. Again, we use the multiplication principle as in (a), but while there were \(n\) options for \(a_1\text{,}\) now there are \(n-1\) options for \(a_2\) because, for the \(a_i\) to be distinct, we cannot repeat, so \(a_{2}\neq a_{1}\) and so there are only \(n-1\) options for \(a_2\text{.}\) Similarly, for \(a_{3}\) there are \(n-2\) options for what it can be (since it can't equal \(a_{1}\) or \(a_{2}\)). Continuing in this way, after having picked \(a_{1},\dots,a_{i-1}\text{,}\) there will be \(n-i+1\) remaining options for \(a_i\text{.}\) Thus, the total number of sequences \(a_{1}, \dots ,a_{k}\) of \(k\) distinct elements of \(S\) is \(n(n-1)\cdots (n-k+1)\text{.}\)
