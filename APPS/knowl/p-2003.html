For the second part, there are \({10\choose 4,4,2}\) ways of picking an ordered partition with sets of sizes \(4,4\text{,}\) and \(2\text{.}\) But what if we instead wanted the number \(N\) of unordered partitions \(\{A_{1},A_{2},A_{3}\}\) where \(|A_{1}|=|A_{2}|=4\) and \(|A_{3}|=2\text{?}\) Given such a partition, there are \(2!\) ordered partitions we can make from it, (by swapping the two sets \(A_1\) and \(A_2\) of the same size \(4\)), so that the first two sets have size 4 and the last has size 2. Thus, counting the number of ordered partitions overcounts the number of unordered partitions by a factor of \(2\text{,}\) so the number of partitions is \(N= {10\choose 4,4,2}/2\text{.}\)
